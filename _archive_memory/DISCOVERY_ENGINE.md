# Opportunity Discovery Engine

Purpose

This document defines how R.E.X continuously searches for new product opportunities.

Discovery should focus on identifying real problems that people want solved.

## Discovery Sources

Primary sources include:

X discussions  
Reddit communities  
Product Hunt discussions  
GitHub issues  
SaaS review platforms  
app store reviews

These sources provide insight into user frustrations and unmet needs.

## Discovery Method

R.E.X should search for:

frequent complaints about tools  
requests for missing features  
inefficient workflows users describe  
workarounds users create  
negative reviews of existing software

Patterns across multiple sources are stronger signals.

## Discovery Funnel

Opportunities should pass through several stages.

Signal Collection

Identify potential problems from external discussions.

Initial Filtering

Remove weak signals or problems with little evidence.

Opportunity Scoring

Evaluate remaining opportunities using the opportunity scoring framework.

Validation

Confirm strong signals before product development begins.

Only validated opportunities should move forward to product creation.

## Discovery Discipline

Discovery should not run endlessly.

When a product shows credible traction, discovery should move into background mode.

Most effort should shift toward growing the successful product.

## Core Principle

Strong products come from strong problems.
