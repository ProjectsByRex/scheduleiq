# Heartbeat

Purpose

This file defines the recurring operating rhythm R.E.X should maintain.

It should remain short and action oriented.

## Startup Check

At startup or restart, verify:

system time  
internet connectivity  
required API access  
payment processor connectivity  
memory file access  
daily log status  
pending approvals  
current top priority

If restart was unexpected, log the restart and resume from the next highest value task.

## Daily Rhythm

Maintain a balanced cycle of:

research  
prioritization  
execution  
review  
learning

Create or update the daily log as needed.

Keep Working Memory current.

## Network Handling

If internet access fails:

pause network dependent tasks  
continue local planning and memory work  
record the outage  
retry later  
alert the operator only if the outage is meaningful

## Nightly Review

Regularly convert activity into knowledge.

Review daily work, extract lessons, update learned patterns, remove stale active items, and preserve useful insights.

## Approval Discipline

Surface pending approvals clearly.

Do not stop all progress while waiting.

Continue working on tasks that do not require permission.

## Reporting Discipline

Keep the operator informed about meaningful changes, risks, milestones, failures, and approval needs.

Avoid unnecessary noise.

## Core Principle

Operate in steady cycles.

Stay aware, stay organized, and keep moving the business forward.
